--------------------------------------------------------------------------------
1. Current Active Pipeline
--------------------------------------------------------------------------------

The active experiment workflow is:

  1. run_experiments.py
     Runs the experiment families and writes CSV outputs into results/

  2. plot_all.py
     Reads the CSV outputs from results/ and generates the final plots

The script comments and the current output folder indicate that the intended
pipeline is:

  results/
    tournament/
    hyper_iterations/
    hyper_rollout_depth/
    hyper_exploration/
    hyper_prior_bonus/

The plot entrypoint is:

  python plot_all.py --results-dir results

--------------------------------------------------------------------------------
2. What run_experiments.py Does
--------------------------------------------------------------------------------

run_experiments.py is designed as an idempotent experiment runner.

Main features:
  - skips any matchup/seed already present in game_records.csv
  - supports resume after interruption
  - writes metadata.json for reproducibility
  - writes ratings.csv and matchup_summary.csv when a run finalizes
  - supports multiprocessing

Two experiment families are implemented:

  1. Tournament
     Intended as the main multi-agent comparison study.

  2. Hyperparameter sweeps
     Focused studies for MCTS-All variants against:
       - Random
       - Heuristic
       - MCTS-Vanilla

Default settings in the current script:
  - seeds: 40
  - base MCTS iterations: 50
  - max turns: 200
  - results dir: results

Hyperparameter sweeps currently implemented:
  - iterations: 25, 50, 100
  - rollout_depth: 4, 8, 12
  - exploration: 0.6, 1.05, 1.4
  - prior_bonus_scale: 1.5, 4.0, 8.0

3. Important Review Finding: Script Comments vs Current Outputs
--------------------------------------------------------------------------------

There is a mismatch between the current code comments/configuration and the
actual completed result files.

In run_experiments.py:
  TOURNAMENT_AGENTS is defined as:
    Random, Heuristic, MCTS-Vanilla, MCTS-All

However, the existing results/tournament/game_records.csv contains six agents:
  - Random
  - Heuristic
  - MCTS-Vanilla
  - MCTS-HeurRollout
  - MCTS-Prior
  - MCTS-All

This means the current tournament results were likely produced by an earlier
or partially modified version of the runner, or the script was changed after
the CSV was generated.

Practical consequence:
  The completed tournament data is still valid as data, but the code comments
  should not be treated as a perfect description of what is already in the
  results folder.
