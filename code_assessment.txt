  MCTS Core (MCTS.py)

  Opponent simulation:
  - Original: Opponents move fully randomly — random.choice(["up","down","left","right"]) — can even move into walls
  - Claude: opponent_joint_responses() evaluates top-2 heuristic moves per opponent, considers all joint combinations (up to 6). More realistic adversary modeling

  UCT / node selection:
  - Original: Standard UCB1 with c=1.4, random tiebreak
  - Claude: PUCT-style — exploit + explore + prior_bonus, where prior_bonus is normalized heuristic score decaying as 18.0 / (1 + visits). Better moves get tried first

  Root scoring:
  - Original: None — tree is built without pre-scoring root moves
  - Claude: opponent_aware_root_scores() computes minimax-style priors at root: 0.55 * worst + 0.30 * average + 0.15 * median across opponent response scenarios

  Rollout (simulation):
  - Original: sim_type == "heuristic" branch is a TODO — both branches are random.choice(moves)
  - Claude: rollout_move() uses local_move_score() with a shortlist (top moves within 18 pts of best) — guided, not pure random

  Safety fallback:
  - Original: None — MCTS result used directly
  - Claude: If MCTS avg < 0 AND heuristic is >60 pts better, overrides MCTS with heuristic choice

  Reward function bug in original:
  # starter-snake-python/MCTS.py:227-230
  for f in state.food:
      food_dist = np.sqrt(...)  # overwrites each loop
      if food_dist < min_dist:
          min_dist = food_dist
  return len(state.you) + state.health / 100 + (5/food_dist)  # uses food_dist, not min_dist!
  Returns distance to the last food in the list, not the nearest.

  ---
  Which is better?

  starter-snake-python-claude is significantly better because:
  1. Opponents aren't random — they play heuristically, so MCTS simulates a more realistic game
  2. PUCT-style priors mean good moves are explored first, not wasted iterations on clearly bad moves
  3. Rollouts are guided by heuristic, not random walks
  4. Root priors account for worst-case opponent responses (minimax blend)
  5. Heuristic safety net prevents catastrophic MCTS misfires
  6. No reward function bug

  How to Run

  starter-snake-python (original)

  cd starter-snake-python
  # Terminal 1 — agent on port 8000
  python main.py

  # Terminal 2 — second agent on port 8001 (same code, different port)
  PORT=8001 python main.py

  # Terminal 3 — run game
  python run_game.py

  starter-snake-python-claude

  cd starter-snake-python-claude

  # Terminal 1
  python mcts_main.py                  # MCTS agent, port 8000

  # Terminal 2
  PORT=8001 python heuristic_main.py   # Heuristic agent, port 8001

  # Terminal 3
  PORT=8002 python random_main.py      # Random agent, port 8002

  # Terminal 4 — pick a matchup
  MATCHUP=mcts_vs_heuristic python run_game.py
  MATCHUP=mcts_vs_random python run_game.py
  MATCHUP=heuristic_vs_random python run_game.py
